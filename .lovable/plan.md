

## Fix: Remove extra spacing in article HTML for Framer

### Problem
The article content generated by Perplexity and stored via TipTap produces HTML with empty `<p></p>` tags and extra whitespace between sections. In the TipTap editor, the `prose` CSS class controls spacing tightly, so it looks fine. But Framer's CMS rich text renderer applies its own default paragraph margins, turning every empty `<p>` into a visible gap.

### Solution
Sanitize the HTML in the `make-webhook` function before sending it to Framer. This is the cleanest approach — it doesn't affect the editor or database, only what Framer receives.

### Changes

**`supabase/functions/make-webhook/index.ts`** — Add an HTML sanitizer function:

```ts
function cleanHtmlForFramer(html: string): string {
  return html
    // Remove empty paragraphs: <p></p>, <p> </p>, <p><br></p>, <p>&nbsp;</p>
    .replace(/<p>\s*(<br\s*\/?>)?\s*<\/p>/gi, '')
    .replace(/<p>&nbsp;<\/p>/gi, '')
    // Collapse multiple consecutive <br> tags into one
    .replace(/(<br\s*\/?\s*>){2,}/gi, '<br>')
    // Remove whitespace-only text nodes between block elements
    .replace(/(<\/(?:p|h[1-6]|div|ul|ol|table|blockquote)>)\s+(<(?:p|h[1-6]|div|ul|ol|table|blockquote))/gi, '$1$2');
}
```

Apply it to the `body` and `faq` fields:
```ts
body: cleanHtmlForFramer(a.content ?? ""),
faq: cleanHtmlForFramer(a.faq_html ?? ""),
```

### Scope
- Single file change: `supabase/functions/make-webhook/index.ts`
- Redeploy the edge function
